# logikalex_vjezba_jedan
upoznavanje s osnovama gita

## 1. Što je Visual Studio Code i za što se najčešće koristi?
- VS Code je program za uređivanje koda i koristi se za programiranje i u raznim projektima
## 2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
- File Explorer
## 3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?
- Idemo na **extensions** panel i tražimo koji želimo, oni služe za dodavanje novih opcija za razne programske jezike i sl.
## 4. Kako otvoriti postojeći projekt ili mapu u VS Code-u?
- **file -> open file ->** otvori nam explorer
## 5. Što se sve može raditi u Exploreru?
- Možemo gledati strukturu projekta, kopirati, brisati datoteke
## 6. Kako možeš brzo pronaći određenu datoteku u projektu?
- **Ctrl + P ->** tražimo file po njegovom imenu
## 7. Kako možeš istovremeno gledati dvije različite datoteke u editoru?
- **Ctrl + Shift + P ->** izaberemo split editor
## 8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?
- **Ctrl + Shift + P ->** format document
## 9. Što je version control i zašto se koristi?
- Koristi se kako bi pratio promjene u kodu i kako bismo mogli otići u prijašnje verzije koda
## 10.  Kako se u VS Code-u otvara panel za Git (Source Control)?
- Sa strane postoji panel za version control
## 11. Što znače commit, push, pull?
- **commit** - mijenjamo verziju projekta koju imamo na računalu
- **push** - "guramo" tu verziju na vanjski repozitorij
- **pull** - uzimamo verziju sa vanjskog repozitorija na svoje računalo
## 12. Čemu služi commit poruka i kako treba izgledati?
- ona služi kako bismo objasnili što smo promjenili, treba bit jasna
## 13. Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?
- Kroz version control panel u VS Code-u ili možemo gledati commitove na GitHub-u
## 14.  Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
- napravimo promjenu, commitamo na računalo, pushamo na repozitorij na GitHub-u
## 15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
- koristimo **git reset**, na GitHub-u možemo otići na stariju verziju
## 16.  Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
- Koristimo **git status**
